# Open Workflow Kit v1.0.1

This corrective GitHub source release makes the Completion Contract integrity check deterministic across Linux, macOS and Windows checkouts.

## Fixed

- Normalize tracked text to LF with `.gitattributes`, preventing Windows `core.autocrlf` from changing Contract-pinned Oracle fingerprints.
- Add an end-to-end regression assertion for LF checkout behavior.
- Include the actual runner blocker detail when the Definition-to-Done end-to-end expectation fails.

## Portfolio presentation

- Reframe the README around team-level AI Coding delivery governance.
- Add a three-project selector, a 30-second dry-run demo, and deterministic Hero, architecture and Social Preview sources.

## Verify

```bash
git clone --branch v1.0.1 --depth 1 https://github.com/bluecoast1379/open-workflow-kit.git
cd open-workflow-kit
npm run check
npm run check:history
```

The release is a GitHub/source release. It does not claim an npm registry publication or real-client certification for the seven generated adapters.

The immutable `v1.0.0` tag remains unchanged; this fix is delivered as the next patch version.
