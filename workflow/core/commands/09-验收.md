# /09-验收

## Goal

由有权签收的人依据 08 验收表完成商业、体验、审美、合规等 human gates，并明确记录 `ACCEPTED`、`REJECTED` 或阻断；不重新判定已由机器证据证明的事实。

## Required Inputs

- `08-验收表格.md`
- `features/{feature}/completion/contract.yaml`、environment/findings manifest、Evidence Ledger、runner run-state 的同一快照
- human attestation key 映射、Owner-controlled Ed25519 ledger-anchor key/public key 与工作区外 anchor 输出位置
- 当前可访问的产品 / 环境、人工验收 rubric、签收角色与授权证据

## Execution Rules

- 只有 `READY_FOR_HUMAN_ACCEPTANCE` 才能进入正式签收；自动项存在 `FAIL/BLOCKED/NOT_RUN/STALE` 时先退回处理。
- Agent 可以组织步骤、记录反馈和校验一致性，不能冒充业务 / 设计 / 合规 Owner 签字。
- Human gate 结论必须 append 到 Evidence Ledger；最后一条 human evidence 后由独立 Owner 在工作区外签发 ledger anchor，再由 `evaluate-dod` 聚合。不得直接编辑 runner run-state 制造 `ACCEPTED`。
- 人工不通过不得被改写为“建议优化”；必须关联原 AC 或提出版本化 contract change。
- 验收授权不等于发布、推送、部署、商店提交或生产写入授权。

## Required Structure

1. **验收快照与参与者**：contract/source/environment/findings 指纹、ledger head/count、日期、签收人、角色和权限范围。
2. **自动完成复核**：READY 状态、blocking AC 聚合、P0/P1 finding、scope drift、豁免与证据新鲜度。
3. **Human gates**：逐个 `AC-###` 的场景、rubric、实际观察、artifact、结论、签收人和时间。
4. **体验直觉**：任务成功、反馈、认知负荷、错误恢复、信任、审美一致、广告 / 劝导克制和 accessibility。
5. **商业与组织判断**：价值假设、运营可行性、依赖承诺、合规审批和上线后指标 Owner。
6. **遗留与豁免**：残余风险、期限、Owner、补偿控制；新的豁免必须遵循 Completion Contract 治理。
7. **签名与聚合**：append human evidence → Owner 签发当前 head/count/fingerprints anchor → evaluator 校验 HMAC、anchor 与全部 blocking gate。
8. **最终结论**：`ACCEPTED / REJECTED / BLOCKED`，适用范围、有效 contract/source/environment/findings、evaluator 输出与下一允许动作。

## Exit Criteria

- 所有 human gate 都有真实、有权 Owner 的 `PASS/FAIL` 结论与证据；没有 Agent 自签或默认通过。
- 最终 `ACCEPTED` 仅在自动项就绪、全部 blocking human gates PASS、findings snapshot 有效且 Owner-signed ledger anchor 对当前 head/count/fingerprints 验证通过时成立。
- `REJECTED/BLOCKED` 明确关联 AC、重现 / 观察、Owner 与返回阶段；不得用笼统“体验不好”替代 rubric。
- 结论没有扩大为发布成功或商业成功；上线后 North Star 仍有独立观测窗口和停止条件。
- 报告、独立 evaluator 输出、runner run-state、状态页和签收记录语义一致；runner run-state 永远保持 `accepted: false`，不得写入 `ACCEPTED`。

## Required Outputs

- `features/{feature}/09-验收.md`
- 工作区外 Owner-signed ledger anchor、独立 evaluator 结果，以及更新后的 `00-工作流状态.md`；不得把 human acceptance 写回 runner run-state
- 需要返工时生成 AC 级 decision / defect packet，不静默改合同。
