# /06-测试用例

## Goal

从 Completion Contract 派生可复现、可证伪的测试与人工验证计划，使每个 blocking `AC-###` 都有 AI 无法回避的 Oracle，而不是另起一套验收标准。

## Required Inputs

- `AGENTS.md`、`workflow/team-profile.yaml`
- 01、02、03、适用的 02B 与由 03 细化、等待 `/define-done` 最终复核的 draft `features/{feature}/completion/contract.yaml`；若是冻结后增补测试，必须先走有权 Owner 批准的 Contract 新版本流程
- 实现存在时读取真实 diff；研发准备阶段则明确标记 `PLAN_ONLY`
- 测试模板、fixture、环境能力和已有回归套件

## Execution Rules

- 测试用例 ID（如 `TC-###`）必须关联一个或多个 `AC-###`；不得用测试用例反向弱化 AC。
- 对照 `workflow/core/checklists/test-blind-spots.md` 十类盲区逐类记录“已覆盖 / N/A + 理由”。
- 同一业务规则的接口、导入、页面、异步、批处理等入口分别设计用例。
- 本阶段设计测试，不执行测试、不修改业务代码；自动化条件成熟时生成 API / UI 计划供 `/07` 执行。

## Required Structure

1. **追踪矩阵**：`REQ/RISK/AC → TC → Oracle → evidence`，标出 blocking、human gate 与覆盖缺口。
2. **可复现上下文**：环境、版本 / commit、依赖、fixture / dataset 版本、随机 seed、账号角色、时钟 / 时区、setup、cleanup 和隔离策略。
3. **功能用例**：正常、异常、边界、空值、权限、多入口、兼容、回归和恢复流程。
4. **一致性与并发**：原子性、幂等、重复 / 乱序、竞态、部分成功、重试、补偿、缓存失效和历史数据。
5. **韧性用例**：弱网、离线、超时、连接 / 内存 / 队列耗尽、限流、熔断、依赖故障、故障注入、RTO/RPO。
6. **性能与成本**：负载模型、预热、样本量、p50/p95/p99、吞吐、容量、资源和单次 / 月度成本的硬阈值与回归容忍度。
7. **安全隐私合规**：鉴权、ACL、越权、恶意输入、秘密泄漏、数据最小化、保留删除、审计、滥用和脆弱用户路径。
8. **UX / Accessibility**：任务成功率、完成时间、反馈延迟、误操作与恢复、键盘 / 屏幕阅读器 / 对比度等自动 Oracle 和人工 rubric。
9. **可观测性与可逆性**：日志 / 指标 / trace / 告警可用性、runbook、flag、kill switch、回滚、数据修复和兼容迁移。
10. **AI 评价（如适用）**：固定黄金集、模型 / prompt 版本、评分器、质量分布、方差、幻觉、安全拒绝、回退和成本；不可只测单个 happy prompt。
11. **反事实与防作弊**：关键断言的 mutation / negative control，证明错误实现会失败；不得删除失败样本或只验证页面可打开。
12. **Flaky 策略**：重跑规则、隔离条件、Owner、到期日；quarantine 不算 PASS。
13. **人工验证**：确实无法自动化的商业、审美、合规和体验判断，提供明确 rubric、证据、签收人和失败条件。

## Exit Criteria

- 每个 blocking `AC-###` 至少有一个确定 Oracle；组合风险有 integration / e2e / fault / performance 等适当层级覆盖。
- 每个用例有前置、步骤、输入、断言、清理、环境 / 数据指纹和证据要求，第三方不可用时有阻断或模拟边界。
- 测试盲区、性能成本、安全隐私、韧性、UX、可观测性、回滚演进与 AI 质量均已覆盖或给出可审计 N/A。
- 存在关键 negative control / mutation 候选，测试能证明错误实现会失败。
- API / UI 自动计划可执行且不含凭证；无法自动化项有人工 rubric 与 Owner。
- 所有计划状态为 `NOT_RUN`；测试设计完成不得被表述为测试通过。

## Required Outputs

- `features/{feature}/06-测试用例.md`
- 适用时生成 `api-test-plan`、`ui-test-plan`、performance / security / AI eval 计划。
- 更新 `completion/contract.yaml` 的 Oracle 引用与 `00-工作流状态.md`，不创建虚假 PASS 证据。
