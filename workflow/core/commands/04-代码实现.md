# /04-代码实现

## Goal

在全部准入闸门通过后，按冻结的 Completion Contract 实现最小且完整的改动，并逐项记录每个 `AC-###` 的实现贡献、风险和可验证事实。

## Required Inputs

- `AGENTS.md`、`workflow/team-profile.yaml`
- 已通过的 01、02、03；有 UI 时读取已通过的 02B
- 冻结且通过定义 lint 的 `features/{feature}/completion/contract.yaml`
- 真实代码基线、实现任务、测试计划与当前 worktree 状态

## Execution Rules

- 只有 feature branch gate、implementation stage gate 与 same-repo parallel gate 全部通过才允许改代码。
- 分支模型和允许路径从 team-profile 与 Completion Contract 读取；不得把模板中的示例分支名当事实。
- 每次改动必须映射 `REQ-###`、`AC-###` 或 `RISK-###`；超出范围先停止并请求 contract change，不得静默扩张。
- 禁止为使检查变绿而降低阈值、删除 / 跳过失败测试、将 `NOT_RUN` 记为 `PASS`、只修改测试适配错误实现、硬编码测试数据或扩大豁免。
- Contract 冻结后 Agent 只能建议变更，不能自行更改目标、blocking 状态、阈值或 human gate。
- 外部写操作仍由 `execution-policy` 控制；实现授权不等于发布授权。

## Required Structure

1. **准入证据**：contract version/hash、branch / base / commit、worktree、dirty 状态、允许 / 禁止路径、闸门结论。
2. **实现映射**：按 `REQ/AC/RISK` 列出真实文件、符号、迁移、配置、接口、UI 和测试改动。
3. **数据与契约**：数据流、状态机、事务、幂等、并发、序列化、错误语义、兼容与迁移的实际实现。
4. **异常与恢复**：弱网、超时、重试、部分失败、资源耗尽、降级、回滚与人工修复入口的实际处理。
5. **质量预算实现**：性能 / 成本控制、安全隐私、可观测性、可逆性、演进 / 下线和 AI 质量机制。
6. **实际变更清单**：代码、配置、数据、依赖、CI/CD、公开材料及未修改项；明确 diff 与计划差异。
7. **本地验证**：格式、lint、类型 / 编译、unit / contract 等已真实运行命令、commit / source fingerprint、结果和证据路径。
8. **残余风险**：未完成 AC、阻断、技术债、假设、人工 gate 与下一验证动作。

## Exit Criteria

- 所有计划内 P0/P1 `REQ/AC/RISK` 都有真实实现映射；无无法解释的 scope drift、死字段、孤立配置或未消费契约。
- 构建 / 静态 / 单元级检查在当前 source fingerprint 上真实通过，或明确记录阻断；计划命令不算证据。
- 异常路径、幂等 / 原子性、性能成本、安全隐私、可观测性、可逆性和兼容迁移已按合同实现。
- 代码没有弱化 Completion Contract、测试或安全边界；新增发现没有被隐藏为 N/A。
- `04-代码实现.md` 与真实 diff 一致；仍为 `NOT_RUN/FAIL/BLOCKED/STALE` 的验收项未被宣布完成。
- 本阶段只证明实现完成，不证明独立审查、全量测试、人工验收或发布完成。

## Required Outputs

- 目标 worktree 中的真实实现与必要自动化测试。
- `features/{feature}/04-代码实现.md`
- 更新 `features/{feature}/00-工作流状态.md`；需要时更新 run-state，但不伪造 Evidence Ledger PASS。
