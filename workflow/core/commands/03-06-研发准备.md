# /03-06-研发准备

## Goal

在已完成 PRD 和必要 UI 设计后，一次性完成“实现前”的技术架构、验证设计与研发准备。该命令不会实现代码、审查不存在的 diff、执行测试或宣称 04/05/07 已完成。

## Required Inputs

- `AGENTS.md`、`workflow/team-profile.yaml`
- `01-需求讨论.md`、`02-产品文档.md`、`features/{feature}/completion/contract.yaml`（draft Completion Contract）
- 有 UI / 前端范围时必须有 `02B-UI设计.md`，除非用户给出范围明确且有期限的设计豁免
- 真实仓库、分支基线、依赖与测试工具链

## Execution Rules

- 严格按 `/03-技术架构` → `/06-测试用例` 的“设计部分”编排；只输出可以在实现前由事实证明的内容。
- 不生成声称真实改动已发生的 `04-代码实现.md`，不生成声称审过真实 diff 的 `05-代码审查.md`，不生成测试通过的 `07-测试执行.md`。
- 可输出 `04-实施计划.md` 与 `05-审查计划.md`，但必须醒目标记 `PLAN_ONLY / NOT_EXECUTED`，且不得更新对应阶段为完成。
- 分支与外部动作按 `execution-policy`；本命令本身不授权代码实现。

## Required Structure

1. **前置闸门检查**：PRD、UI 闸门、Completion Contract、仓库事实源、分支 / worktree 和阻断依赖。
2. **技术架构**：完整执行 `/03-技术架构` 的 Required Structure 与 Exit Criteria。
3. **实施计划**：按 `REQ/AC` 拆解任务、路径、依赖、顺序、风险、验证点与允许范围；状态固定为 `PLAN_ONLY`。
4. **审查计划**：按风险指定独立检查面、所需 diff / runtime evidence 与阻断规则；状态固定为 `PLAN_ONLY`。
5. **测试设计**：完整执行 `/06-测试用例` 的测试设计要求，生成尚未执行的 Oracle 与计划；所有用例状态为 `NOT_RUN`。
6. **准入摘要**：可进入实现 / 阻断、阻断理由、下一允许命令和需要用户决策的 decision packet。

## Exit Criteria

- `/03-技术架构` 的 Exit Criteria 全部满足，Completion Contract draft 与全部 `NOT_RUN` Oracle 已达到可由 `/define-done` 最终复核和冻结的状态。
- 每个 blocking `AC-###` 已映射至少一个可执行或人工 Oracle；测试计划明确为 `NOT_RUN`。
- 实施任务和审查计划覆盖全部 P0/P1 `REQ/RISK/AC`，没有把计划冒充执行证据。
- 有 UI 范围时 02B 已通过或存在合规豁免；分支、worktree、依赖、权限和测试环境无未声明歧义。
- 状态文件只把 03 与 06 的“设计完成”写为相应状态；04/05/07 仍为未执行或未授权。

## Required Outputs

- `features/{feature}/03-技术架构.md`
- `features/{feature}/04-实施计划.md`（`PLAN_ONLY`，不是 `04-代码实现.md`）
- `features/{feature}/05-审查计划.md`（`PLAN_ONLY`，不是 `05-代码审查.md`）
- `features/{feature}/06-测试用例.md` 与适用的可执行测试计划（全部 `NOT_RUN`）
- 更新 draft `completion/contract.yaml` 与 `00-工作流状态.md`；下一允许命令是 `/define-done`，不是代码实现。
