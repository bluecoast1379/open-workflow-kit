# /05-代码审查

## Goal

代码审查: 以问题优先方式审查真实 diff、发布边界、PRD 一致性和残余风险。

## Required Inputs

- `AGENTS.md`
- `workflow/team-profile.yaml`
- Previous stage documents under workspace-level `features/{feature}/`
- Frozen `features/{feature}/completion/contract.yaml` (Completion Contract)
- Local code, local docs, and user-provided source materials listed in team-profile

## Execution Rules

- Read local facts before writing conclusions.
- Distinguish verified facts, design intent, assumptions, and missing evidence.
- 默认使用简体中文展示工作流沟通和阶段产物；专有名词、产品名、品牌名、代码标识符、命令、文件路径、分支名、API、SDK、框架、协议、标准、错误信息和官方英文术语保留原文。
- Do not claim tests, builds, screenshots, deployments, or reviews passed unless they were actually executed.
- Git、数据库、部署、生产配置和外部写操作按 `workflow/core/execution-policy.md` 的 core 上限、外部受信策略、仓库请求与当次授权四层最严格结果执行；审查阶段不扩大授权。
- This stage does not authorize business code changes unless the current command is an implementation command and all gates pass.

- Findings lead the output. Order issues by severity and cite files, diffs, tests, or runtime evidence.

## 强制检查清单

按 `workflow/core/checklists/` 逐单核对，并在审查文档记录每单"命中 / 不适用 / 未核查原因"：

- `branch-hygiene.md`：分支流向（只出不进）、发布候选溯源、共享制品版本、回归恢复守护。
- `validation-change-review.md`：本期涉及放开/收紧任何校验或不变式时必查。
- `data-consistency-review.md`：涉及统计口径、时间列、导出、手工 SQL 时必查。
- `third-party-integration-review.md`：涉及第三方接口、消息消费、变更扇出时必查。
- `test-blind-spots.md` §B：关键路径（金额/契约/共享组件/口径变更）必须做独立对抗式复审，含死字段检测（对新增字段 grep setter 与写入点，零写入 = 未实现缺陷，不得写成"待测试"）。
- `language-pitfalls-java.md`：技术栈含 Java 时必查。

## Required Structure

1. 审查基线：contract/hash、真实 diff range、branch / base / source fingerprint、生成物与未跟踪文件。
2. Findings-first：按 P0–P3 排序，给出 file:line / runtime evidence、违反的 `AC/REQ/RISK`、影响与修复建议。
3. 范围与反作弊：scope drift、测试 / threshold 弱化、死字段、硬编码、豁免和 Evidence Ledger 异常。
4. 跨层契约：入口到终点、数据 / 状态 / 错误、兼容 / 迁移、provider / consumer 和多入口一致性。
5. 质量维度：业务规则、UX、性能成本、韧性、安全隐私、可观测性、可逆演进、AI 质量与供应链。
6. 检查清单结果、实际运行验证、残余风险和无法核查证据；独立 verifier 身份 / 方法。

## Exit Criteria

- 审查覆盖完整真实 diff 与未跟踪范围，不以 PR 描述或实现者总结代替事实。
- 无开放 P0/P1 finding；P2/P3 有 Owner、期限和不阻断理由。
- 每个 blocking AC 的实现映射、异常路径、质量预算和 anti-cheating 均已核查。
- 所有强制清单记录命中 / N/A + 理由 / 未核查阻断，独立审查不自签无法验证项。
- 审查结论绑定 source fingerprint；代码变化后受影响结论失效。

## Required Outputs

- Update or create the corresponding file under workspace-level `features/{feature}/`.
- Update workspace-level `features/{feature}/00-工作流状态.md` when stage status changes.
- Record unresolved questions and evidence gaps explicitly.
