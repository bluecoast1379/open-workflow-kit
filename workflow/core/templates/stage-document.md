### 不允许的动作

- 未经用户手动操作，不执行远程 Git 刷新、创建分支、推送、构建部署、数据库写入或生产配置写入。
- 未通过实现准入，不修改业务代码、配置、脚本、迁移或部署文件。
- 不将工作流阶段产物、PRD、调研、法务、隐私、合规、验收、复盘或内部决策材料写入目标代码仓库；这些材料应保存在工作区级 `features/{feature}/`。
- 不在 `main` 或 `prod` 上直接开发；实现必须在从 `prod` 创建的开发分支上进行，完成后合并到 `main` 测试，再由 `main` 上线到 `prod`；个人项目不创建 `test` 分支。
- 首个线上版本前不启用自动生产部署；CI/CD 变更必须记录触发分支、必跑检查、密钥存放、部署审批、回滚路径和证据。
